Backend Templating

Simple applications can be built using just native HTML markup and custom HTML5 Web Components such as metaphactory's semantic components for search, data visualization, and data curation (see below). Nevertheless, in some situations it might be desired to modify or control the template- or application pages dynamically before they leave the backend and are rendered in the client's browser. Use cases include:
  • dynamic parameterization of the HTML components with backend knowledge
  • excluding fragments according to the availability of data or security restrictions (especially when defined on generic templates)
  • to outsource certain fragments into separate, reusable units in favor of maintainability
Therefore, template and application pages can be enriched with a "mostly logic-less" backend templating logic such as, for example, using if expressions or include helper functions.
