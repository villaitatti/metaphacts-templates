Metaphactory Components

Beyond React Bootstrap, the metaphactory exposes some additional 3rd party react components or own implementations of components for different purposes.
All components provided directly or indirectly (e.g. by small wrappers) through the metaphactory are being prefixed with mp-*.

Semantic / Resource Link

The is aliased and registered under as well.

This special component resolves links for navigation purpose within the platform i.e. even though a IRI might be different then the physical location of the metaphactory installation, the link component will route all navigation requests to stay within the platform.

Modal Dialog / Overlay

The to open some content in a modal dialog or overlay. A dialog is just a variant of on overlay. By default the attribute type is set to type=modal. Setting it to type=lightbox will create an lightbox overlay instead of modal dialog. Arbitrary elements can be wrapped into element to open (trigger) the dialog/overlay showing any elements specified within element.

Page Loader

The component behaves similar to a server-side page include with the difference that the loading and rendering is done asynchronously on client side. The requested page is compiled in backend with the context of the current resource (same as an include).
