Installation and Deployment

Deployment Modes

For the time being three deployment artifacts are available:
  • Docker Image - Recommended and well tested deployment mode. Easy to update and migrate once new releases are available. Takes just a few minutes to login to metaphacts registry, pull the platform image and start the container.
  • ZIP Artifact - No external dependencies. Simply unzip and run the start script (batch or shell). Recommended for quick getting started or for constrained environments. No automated upgrade procedures.
Please contact support@metaphacts.com in order to request access to downloads and/or credentials for docker registry.

Docker Deployment

Within the docker environment, the platform and Blazegraph™ will run in two separate docker containers.
  1. Install Docker (see official documentation)
  2. Login into Metaphacts docker registry: docker login docker.metaphacts.com
  3. Pull latest platform and Blazegraph™ image
    • docker pull docker.metaphacts.com/release/blazegraph:2.2.0
    • docker pull docker.metaphacts.com/release/platform-data:2.0.0
    • docker pull docker.metaphacts.com/release/platform:2.0.0
  4. Prepare & run Blazegraph™ container
    • Create data (journal) container for Blazegraph™ container: docker create --name blazegraph-journal -v /blazegraph-data ubuntu
    • Run Blazegraph™ container with data container mounted: docker run --name blazegraph -d --restart=always -p 10080:8080 --env JAVA_OPTS="" --volumes-from blazegraph-journal docker.metaphacts.com/release/blazegraph:2.2.0
  5. Prepare & Run Platform container
    • Create platform data container from image: docker create --name platform-data docker.metaphacts.com/release/platform-data:2.0.0
    • Run platform container with data container mounted: docker run --name platform -d --restart=always -p 10214:8080 --link blazegraph:blazegraph --volumes-from platform-data --env PLATFORM_OPTS="-Dconfig.global.homePage=Help:Start -Dconfig.environment.sparqlEndpoint=http://blazegraph:8080/blazegraph/sparql -Dcom.metaphacts.config.baselocation=/config -DruntimeDirectory=/ -Dconfig.environment.appsDirectory=/apps" docker.metaphacts.com/release/platform:2.0.0
  6. Open http://dockerip:10214/ in your browser (i.e. run 'boot2docker ip' to get IP from boot2docker)

ZIP Artifact

  1. Unzip artifact
  2. Unix Only: Make start.sh executable chmod +x start.sh
  3. Start platform: sh start.sh (Unix) or doubleclick start.bat (Windows)
  4. Open http://localhost:10214/ in your browser
  5. Login with admin/admin.
    Please Note: It is highly recommended to change the password after your first successful login (otherwise you will forget to change it later). The help page basic system configuration describes in detail how to change the standard login.
  6. Please refer to README.txt (located in root folder of the artifact) for further information.
